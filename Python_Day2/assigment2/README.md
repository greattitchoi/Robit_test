# 지뢰찾기 (Minesweeper) — pygame

외부 라이브러리 **pygame**을 활용하여 구현한 클래식 지뢰찾기 게임입니다.
`pip`로 설치한 외부 라이브러리의 창 생성 / 이벤트 처리 / 2D 렌더링 기능을 사용해,
표준 라이브러리만으로는 만들기 어려운 실시간 GUI 애플리케이션을 구성했습니다.

## 2. 설치 및 실행 방법

```bash
# 1) (권장) 가상환경 생성 및 활성화
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 2) 외부 라이브러리 설치
pip install pygame              # 또는: pip install -r requirements.txt

# 3) 설치된 버전 확인
python -c "import pygame; print(pygame.version.ver)"
# -> 2.6.1

# 4) requirements.txt 생성 (본 저장소에는 이미 포함되어 있음)
pip freeze > requirements.txt

# 5) 실행
python main.py
```

실행하면 콘솔에 사용 중인 pygame 버전이 출력된 뒤 게임 창이 열립니다.

## 3. 조작법

| 입력 | 동작 |
|---|---|
| **좌클릭** | 칸 열기 (열린 숫자 칸을 누르면 코딩 동작) |
| **우클릭** | 깃발 표시 / 해제 |
| **스마일 버튼** / `R` | 재시작 |
| `1` / `2` / `3` | 난이도 변경 (초급 9×9·지뢰 10 / 중급 16×16·40 / 고급 30×16·99) |
| `ESC` | 종료 |

상단 패널에는 **남은 지뢰 수(전체 지뢰 − 꽂은 깃발)** 와 **경과 시간(초)** 이 표시됩니다.

## 4. 코드 설명

전체 구조는 **데이터(게임 규칙)** 와 **표현(화면·입력)** 을 분리한 2개 클래스로 구성했습니다.
이렇게 나누면 게임 규칙을 pygame 없이도 단독으로 테스트할 수 있습니다.

### `Board` 클래스 — 게임 로직 (pygame 의존성 없음)

칸의 상태를 4개의 2차원 리스트로 병렬 관리합니다.

| 속성 | 의미 |
|---|---|
| `mine[r][c]` | 지뢰 여부 |
| `number[r][c]` | 주변 8칸의 지뢰 개수 |
| `revealed[r][c]` | 열린 칸 여부 |
| `flagged[r][c]` | 깃발 여부 |

주요 메서드:

- **`place_mines(safe_r, safe_c)`** — 지뢰를 **첫 클릭 직후**에 배치합니다.
  첫 클릭 칸과 그 주변 8칸을 후보에서 제외한 뒤 `random.sample()`로 중복 없이 뽑기 때문에,
  **첫 클릭에서 즉사하는 일이 없습니다**(first-click safety). 배치가 끝나면 모든 칸의 `number`를 미리 계산해 두어,
  렌더링할 때마다 주변을 다시 세지 않도록 했습니다.

- **`reveal(r, c)`** — 칸을 엽니다. 지뢰면 즉시 `LOSE`.
  주변 지뢰가 0인 빈 칸이면 **BFS(너비 우선 탐색)** 로 인접한 빈 칸들을 연쇄적으로 펼칩니다.
  재귀 대신 `stack` 리스트로 구현해 고급 난이도(30×16)에서도 재귀 깊이 초과가 발생하지 않습니다.

- **`chord(r, c)`** — 열린 숫자 칸 주변의 깃발 수가 그 숫자와 일치하면 나머지 칸을 한 번에 엽니다.

- **`check_win()`** — `열린 칸 수 == 전체 칸 수 − 지뢰 수`가 되면 승리 처리합니다.
  (지뢰를 제외한 모든 칸을 열면 이기는 규칙)

### `Game` 클래스 — pygame 화면 및 이벤트 루프

- `pygame.display.set_mode()` 로 난이도에 맞춰 창 크기를 동적으로 계산해 생성
- `pygame.event.get()` 으로 마우스/키보드 이벤트 수집 → `Board`의 메서드 호출
- `cell_at()` 은 마우스 **픽셀 좌표 → (행, 열) 인덱스** 변환을 담당 (`(x - MARGIN) // CELL`)
- `draw_bevel()` / `draw_mine()` / `draw_flag()` 는 `pygame.draw`의 도형 함수만으로 그렸습니다.
  → **외부 이미지 리소스 파일 없이 `main.py` 단일 파일로 실행 가능**
- `pygame.time.Clock().tick(60)` 으로 프레임을 60 FPS로 고정

### 메인 루프의 흐름

```
while True:
    이벤트 처리  (입력 → Board 상태 변경)
    화면 지우기  (screen.fill)
    그리기       (패널 → 보드 → 결과 오버레이)
    화면 갱신    (display.flip)
    프레임 제한  (clock.tick)
```

## 5. 라이브러리에 대한 이해

### pygame이란?

pygame은 멀티미디어 처리 라이브러리인 **SDL(Simple DirectMedia Layer)** 을 파이썬에서 쓸 수 있도록 감싼 래퍼(wrapper) 라이브러리입니다.
C로 작성된 SDL이 OS별 그래픽·입력·사운드 API를 추상화해 주고, pygame은 이를 파이썬 객체로 노출합니다.
덕분에 창 생성이나 마우스 이벤트 처리 같은 저수준 작업을 OS에 상관없이 동일한 코드로 처리할 수 있습니다.

### 이번 프로젝트에서 실제로 사용한 기능

| 모듈 | 사용처 |
|---|---|
| `pygame.display` | 창 생성(`set_mode`), 화면 갱신(`flip`), 제목 설정 |
| `pygame.event` | 마우스 클릭 / 키보드 / 종료 이벤트 큐 처리 |
| `pygame.draw` | 사각형·원·선·다각형·호를 이용한 칸, 지뢰, 깃발, 스마일 렌더링 |
| `pygame.font` | 칸의 숫자, 남은 지뢰 수, 타이머 텍스트 렌더링 |
| `pygame.Rect` | 칸 영역 표현 및 클릭 판정(`collidepoint`) |
| `pygame.time` | `Clock.tick()`으로 FPS 고정, `get_ticks()`로 경과 시간 측정 |
| `pygame.mouse` | `get_pressed()`로 좌·우 동시 클릭(코딩) 감지 |

### 왜 pygame을 선택했는가

1. **표준 라이브러리로는 불가능한 영역** — 파이썬 기본 GUI인 tkinter로도 지뢰찾기는 가능하지만,
   tkinter는 위젯 기반이라 매 프레임 화면을 다시 그리는 게임 루프 구조와는 맞지 않습니다.
   pygame은 픽셀 단위 렌더링과 프레임 루프를 전제로 설계되어 있어 게임에 훨씬 적합합니다.
2. **의존성이 가볍고 설치가 간단함** — `pip install pygame` 한 줄이면 되고, 휠(wheel) 배포판이라 별도 컴파일이 필요 없습니다.
3. **외부 리소스 없이 완결 가능** — `pygame.draw`만으로 모든 그래픽을 그릴 수 있어 제출물이 단일 `.py` 파일로 유지됩니다.

### 왜 2.6.1 버전인가

`pip install pygame` 시 설치되는 최신 안정 버전입니다.
pygame 2.x는 SDL2 기반으로, SDL1을 쓰던 1.9.x 대비 고해상도 디스플레이·창 관리·성능이 개선되었습니다.
버전을 `requirements.txt`에 `pygame==2.6.1`로 **고정(pinning)** 하여, 다른 환경에서도 동일한 동작을 재현할 수 있도록 했습니다.

---

## 6. 배운 점

- **로직과 렌더링을 분리**하면 게임 규칙(`Board`)을 pygame 창 없이도 단위 테스트할 수 있습니다.
  실제로 첫 클릭 안전성, 숫자 계산, BFS 연쇄 열기, 승/패 판정을 창을 띄우지 않고 검증했습니다.
- 지뢰를 **초기화 시점이 아니라 첫 클릭 직후에 배치**하는 것이 지뢰찾기의 핵심 설계 포인트였습니다.
- 게임 루프는 `이벤트 → 상태 갱신 → 렌더링 → 프레임 제한`의 반복이며,
  이 구조는 pygame뿐 아니라 대부분의 게임 엔진에 공통으로 적용되는 패턴입니다.
